### How It Works

1. **Language Selection**: User clicks language button in top navigation
2. **State Update**: `setLanguage('no')` updates context state
3. **localStorage Sync**: Choice saved to `snakke-language` key
4. **Re-render**: All components using `t()` or `tIcon()` re-render with new language
5. **No Route Change**: URL stays `/communicate` (not `/en/communicate`)

### Usage in Components

```tsx
import { useLanguage } from '@/contexts/LanguageContext';

export default function MyComponent() {
  const { language, setLanguage, t, tIcon } = useLanguage();
  
  return (
    <div>
      {/* Translate UI text */}
      <h1>{t('communicate.title')}</h1>
      <button>{t('sentence.speak')}</button>
      
      {/* Translate icon names */}
      <span>{tIcon('water')}</span> {/* Water or Vann */}
      
      {/* Check current language */}
      {language === 'en' ? 'English UI' : 'Norwegian UI'}
    </div>
  );
}
```

## âš ď¸Ź Critical: next-intl Issues

### Why We DON'T Use next-intl

**History**: We attempted to implement i18n using the popular `next-intl` library with Next.js 15 App Router.

**Timeline**: 200+ chat messages debugging routing errors, 404s, and build failures.

**Result**: Complete failure. Had to revert all changes.

### Specific Problems Encountered

#### 1. Routing Complexity
```
Required structure:
/app/[locale]/page.tsx
/app/[locale]/communicate/page.tsx
/app/[locale]/dashboard/page.tsx

URLs became:
/en/communicate
/no/communicate

Problem: 404 errors, route conflicts, navigation breaks
```

#### 2. Middleware Configuration
```typescript
// middleware.ts
import createMiddleware from 'next-intl/middleware';

export default createMiddleware({
  locales: ['en', 'no'],
  defaultLocale: 'en'
});

Problem: Middleware blocked routes, caused redirects, conflicted with auth
```

#### 3. Server Component Issues
```tsx
// This pattern failed repeatedly:
import { useTranslations } from 'next-intl';

export default async function Page() {
  const t = useTranslations('communicate');
  return <h1>{t('title')}</h1>;
}

Error: "useTranslations is not available in Server Components"
```

#### 4. Build Failures
```
Module not found: Can't resolve 'next-intl/server'
TypeError: Cannot read properties of undefined (reading 'locale')
Error: Locale not found in URL
```

#### 5. Cache Corruption
- `.next` folder had to be deleted repeatedly
- Build artifacts caused persistent errors
- Fresh builds still failed

### Why Client-Side Works Better

âś… **Advantages**:
- No routing changes (URLs stay `/communicate`)
- No middleware needed
- Instant language switching (no page reload)
- Simple implementation (1 context file)
- localStorage persistence
- Works with the current Next.js 16 App Router implementation
- No build issues
- Perfect for AAC apps (children need quick language changes)

âťŚ **Trade-offs**:
- Translations loaded client-side (but they're small)
- No SEO benefit from locale URLs (not needed for authenticated PWA)
- No server-side translation (not needed for our use case)

### Attempted Solutions That Failed

1. âś— Different middleware configurations
2. âś— Moving files to `[locale]` folder structure
3. âś— Using `getRequestConfig` from next-intl
4. âś— Setting up locale detection
5. âś— Configuring `i18n.ts` routing
6. âś— Using `next-intl/server` vs `next-intl`
7. âś— Fresh installs and cache clearing
8. âś— Different Next.js versions

### Final Decision

**Architecture**: Client-side React Context i18n
**Reason**: Reliable, simple, perfect for our AAC app use case
**Warning**: DO NOT attempt next-intl with Next.js 15 App Router again

## Maintenance

### Keep Translations in Sync

When adding new features:

1. Add English translation key to `translations.en`
2. Add Norwegian translation to `translations.no`
3. Add translations for any other languages
4. Use `t('key')` or `tIcon('id')` in components

### Testing New Translations

1. Switch to Norwegian (đź‡łđź‡´ NO)
2. Navigate to all pages
3. Verify all text is translated
4. Check speech synthesis uses Norwegian
5. Refresh page - language should persist

### Common Issues

**Problem**: New text not translating
**Solution**: Check translation key exists in both `en` and `no` dictionaries

**Problem**: Language not persisting after refresh
**Solution**: Check localStorage is not blocked, verify `snakke-language` key exists

**Problem**: Icon names showing IDs instead of names
**Solution**: Use `tIcon(icon.id)` not `icon.name`
